  • headerOffset ∷ !Word16

    The offset within the binaryBlob at which the header starts.

  • headerSize ∷ !Word16

    How many bytes the header is long. Extracting the headerSize bytes from binaryBlob starting from headerOffset should yield the header.

data BlockComponent db a where Source #

Which component of the block to read from a database: the whole block, its header, its hash, the block size, ..., or combinations thereof.

NOTE: when requesting multiple components, we will not optimise/cache them.
