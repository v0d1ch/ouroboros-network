This function runs forever unless an exception is thrown.

  • hfbtMaxClockRewind ∷ NominalDiffTime

    Maximum time the clock can be rewound without throwing a fatal SystemClockMovedBack exception.

    When the slot length is short, e.g., Praos' 1s compared to PBFT's 20s, the chances of an NTP sync causing the clock to go back to the previous slot increase.

    We allow the system clock to rewind up to hfbtMaxClockRewind, tracing a TraceSystemClockMovedBack message in such cases. Note that the current slot *never decreases*, we just wait a bit longer in the same slot.
